기존 블로그 글 — 어떻게 접근하고 싶으세요?

Git에 발행된 글을 바로 편집하는 방식에는 두 가지 접근이 있어요. 현재 흐름과 개선안을 비교해드릴게요.

현재 방식 — 수동 "가져오기"
포스트 라이브러리
도쿄 여행 가이드
/tokyo-guide/
● 발행됨
신주쿠 맛집 특집
/shinjuku-food/
● 발행됨
⚠️ 가져오기를 해야만 편집 가능.
매번 확인 팝업이 뜨고 클릭이 두 번 필요함.

도쿄 여행 가이드
/tokyo-guide/
신주쿠 맛집 특집
/shinjuku-food/
⚠️ 글이 많아지면 페이지 로드가 느려질 수 있음.
DB에 중복 데이터가 쌓임. 관리가 복잡해질 수 있음.
🔧 기술 설명:
  • A (추천): "편집" 클릭 시 API가 Git에서 글 읽어와 DB에 저장 → 에디터 오픈. 사용자는 차이 모름. 빠름.
  • B: 페이지 열릴 때 모든 글을 한꺼번에 DB에 복사. 글이 많아질수록 느려지고 DB 낭비.
  • 현재 가져오기 API (/admin/api/posts/[slug]/import/)는 이미 있어요. A는 이걸 자동으로 호출하는 것뿐.
