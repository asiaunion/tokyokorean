# 📑 도쿄 부동산 투자 가이드 수치 검증 및 링크 고도화 기획서 (Research & Planning Document)

이 문서는 GSF-Blog의 『도쿄 부동산 투자 완전 가이드 2026』 포스트 내 인용 수치를 1차 사료와 비교 검증하고, 외부 통계 링크의 구조적 문제점(Link Rot 및 UX 장벽)을 해결하기 위한 구체적인 기술적 방향성과 실행 계획을 담고 있습니다. 다른 에이전트(Cursor 등)가 이 문서를 읽고 즉각적으로 검증 및 고도화 작업을 수행할 수 있도록 작성되었습니다.

---

## 1. 대화 맥락 및 핵심 목표
*   **배경**: 『도쿄 부동산 투자 완전 가이드 2026』 포스트에 여러 부동산 통계 수치와 출처 링크가 있으나, 링크를 클릭해도 세부 통계가 명시된 구체적인 페이지나 PDF로 연결되지 않고 메인 홈으로만 이동하여 독자의 1차 검증이 어려움.
*   **목표**: 
    1.  인용된 모든 수치의 진위 여부를 1차 사료와 전수 대조하여 검증함.
    2.  단순히 외부 메인 도메인을 링크하는 문제를 극복하고, 독자가 즉각 검증할 수 있으면서도 절대 깨지지 않는(영구 보존성) 링크 연동 메커니즘을 수립함.

## 2. 1차 사료 대조 및 수치 정밀 검증 결과 `[1차 확인]`

## 3. 기존 링크의 문제점 및 기술적 극복 장벽

### ❌ 1. 링크 깨짐 (Link Rot) 및 도메인 개편 오류
*   **현상**: 국토교통성의 조직 개편으로 보도자료 URL 경로가 기존 `totikensangyo02`에서 `tochi_fudousan_kensetsugyo05`로 변경되어 기존 글의 주소가 **404 에러**를 뿜고 있었음. 미키상사의 특정 PDF 경로는 `dcms-error` 페이지로 자동 리다이렉트되어 오류 발생.
*   **분석**: 일본 부동산 및 공공 통계 기관들은 매월 새로운 통계가 나오면 이전 달의 PDF나 주소를 아카이브 폴더로 숨기거나 임의로 폐기합니다. 따라서 단순 외부 딥링크 하드코딩은 유지보수가 원천적으로 불가능합니다.

### ❌ 2. 독자의 검증 피로도 (UX 장벽)
*   **현상**: 깨지지 않도록 하기 위해 레인즈(`reins.or.jp`) 또는 칸테이(`kantei.ne.jp`) 등의 메인 포털 홈페이지만 링크할 경우, 독자가 수치 검증을 하려면 수십 개의 하브 메뉴를 타고 들어가 특정 연월의 리포트 PDF를 일일이 대조해야 함. 이는 독자에게 불신을 주거나 검증을 포기하게 만드는 심각한 UX 실패입니다.

## 4. 고도화 방향성 설계 및 구현 시나리오

커서와 에이전트가 향후 GSF-Blog를 고도화할 때 실행할 수 있는 **2가지 구체적인 테크니컬 아키텍처**입니다.

### 🎯 [1안] 아카이브 & 자체 호스팅 전략 (Local Archiving)
외부 통계 사이트들의 URL 유동성을 극복하기 위해, 블로그 빌드 시점 혹은 통계 발표 시점에 **1차 출처 원본 PDF 파일들을 블로그 내부에 직접 영구 보존(Archive)하고 서빙**하는 최선책입니다.

*   **구조 설계**:
    ```
    GSF-Blog Monorepo
    ├── public/
    │   └── assets/
    │       └── sources/            <-- 1차 통계 자료 아카이브
    │           ├── mlit-202511-mansion-report.pdf
    │           ├── reins-202604-marketwatch.pdf
    │           └── kantei-202604-chugomansion.pdf
    └── src/
        └── data/
            └── blog/
                └── ko/
                    └── tokyo-real-estate-investment-complete-guide.md (본문)
    ```
*   **본문 마크다운 적용 방식**:
    ```markdown
    국외 거주자의 도심 6구 신축 취득 비중은 **7.5%**로 나타났습니다.
    ([국토교통성 2025년 11월 공식 보고서(PDF)](/assets/sources/mlit-202511-mansion-report.pdf))
    ```
*   **기대 효과**: 외부 도메인이 리뉴얼되거나 PDF가 사라져도 100% 무중단으로 작동하며, 클릭 시 0.1초 만에 정확한 통계 원본 문서가 화면에 뿌려집니다.

---

### 🎨 [2안] 블로그 내 '1차 자료 팝업 뷰어 (PDF Modal)' 컴포넌트 임베딩
독자가 페이지를 이탈하여 PDF를 다운로드할 필요도 없이, 본문 수치 옆에 **[1차 확인]** 미니 아이콘을 제공하고 이를 누르면 **브라우저 레이어 상에 팝업창(Modal)으로 해당 PDF 문서의 정확한 페이지가 정밀 포커싱되어 열리는 인터랙티브 UX**입니다.

*   **기술 스택**: Astro / React Component + `react-pdf` 또는 browser-native iframe embed.
*   **기능 요건**:
    *   사용자가 `[1차 확인]` 아이콘을 호버하거나 클릭 시 모달 윈도우 생성.
    *   PDF 문서의 전체 페이지가 아니라, 성약 단가 표가 위치한 **특정 페이지(예: page=2)가 기본 포커스**되어 렌더링되도록 구현.
*   **기대 효과**: 일반 블로그들과 궤를 달리하는 압도적으로 프리미엄하고 인터랙티브한 E-E-A-T 전문성(Tier 3 Design Aesthetics)을 독자에게 선사합니다.
