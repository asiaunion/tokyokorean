## 💬 3단계: 근본적인 UX 페인 포인트 해결에 대한 논의

### 👤 User
> 예를 들어 <레인즈 데이터 라이브러리 바로가기> 이 링크는 https://www.reins.or.jp/pdf/trend/mw/mw_202604_summary.pdf 이 페이지를 의미하는 것으로 보이는데, 실제 <레인즈 데이터 라이브러리 바로가기> 링크를 누르면 해당 페이지는 안나옴. 또 <도쿄 칸테이 공식 홈페이지 바로가기>링크는 https://www.kantei.ne.jp/wp-content/uploads/c202604.pdf 페이지를 의미하는 것으로 보이는데, 역시 해당 링크를 클릭하면 일반적인 사이트 화면으로 연결됨(실제 데이터를 찾고 들어가기가 어려움 - 나도 번거롭고 시간이 걸리는데 일반 독자들은 더 어려울 것임). 어떻게 해결/개선 가능할지?

### 🤖 Antigravity (AI)
> 날카로운 지적에 100% 공감합니다. 작성자 입장에서 `mw_202604_summary.pdf` 같은 특정 월의 구체적인 PDF 주소를 글에 직링크해두면 다음과 같은 **기술적 한계**가 발생합니다:
> 1. **링크 깨짐 (Link Rot)**: 다음 달 통계가 발표되는 순간, 외부 통계 기관들은 이전 달의 PDF 링크를 서버에서 가차 없이 지우거나 아카이브로 돌려버려, 블로그 링크가 순식간에 404로 깨집니다.
> 2. **사용자 경험(UX) 실패**: 이를 피하기 위해 안전하게(?) `reins.or.jp` 메인 화면만 걸어두면, 사용자는 수많은 탭을 헤매며 1차 사료를 직접 찾아야 하므로 신뢰도 검증을 포기하게 됩니다.
> 
> 이를 해결하기 위한 **2가지 모던 웹 아키텍처 개선 방안**을 제안합니다:
> 
> * **[1안] 아카이브 & 자체 호스팅 전략 (Local Archiving) [강력 추천]**:
>   * 외부 서버에 독자를 보내는 대신, 통계 발표 시점에 1차 출처 PDF 원본을 블로그 프로젝트 내부(`public/assets/sources/`)로 다운로드하여 자체 서빙하는 방식입니다.
>   * *효과*: 외부 도메인이 리뉴얼되어 파일이 지워져도 100% 정상 작동하며, 독자는 링크를 클릭하자마자 0.1초 만에 우리 서버에서 호스팅하는 정확한 PDF 문서를 볼 수 있어 UX와 신뢰도가 극대화됩니다.
> * **[2안] PDF 팝업 뷰어(PDF Modal) 임베딩**:
>   * 본문 수치 옆에 `[1차 확인]` 미니 아이콘을 두고, 호버나 클릭 시 브라우저 내부에서 팝업창(Modal)으로 해당 아카이브 PDF의 **정확한 페이지(예: 2페이지 성약 단가 표)**를 즉시 렌더링해주는 인터랙티브 기능입니다.
>   * *효과*: 독자가 페이지를 이탈할 필요 없이 그 자리에서 수치를 대조할 수 있는 프리미엄 브랜드 UX를 선사합니다.
