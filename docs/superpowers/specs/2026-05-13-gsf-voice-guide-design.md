# GSF Voice Guide — 설계 문서

**작성일**: 2026-05-13  
**적용 범위**: GSF-Blog 전체 포스트 (KO 원고 작성 + EN/JA 번역)  
**상태**: 승인됨

---

## 1. 배경 및 목적

현재 GSF-Blog의 KO 원고는 전문 용어 설명이 없고 문장이 길어 독자가 막히는 지점이 생긴다.
EN/JA 번역도 KO의 문제를 그대로 굳혀버려 자연스럽지 않다.

목표:
- KO: 뉴스레터 톤 + 저자 개인 경험·의견이 드러나는 방식으로 품질 향상
- EN/JA: voice-consistent 번역으로 원문 뉘앙스 보존

---

## 2. 타깃 독자

투자 경험이 있는 한국인 30~40대. 전문 용어를 아예 모르진 않지만, 설명 없이 쓰면 흐름이 끊긴다.
A타입(실질 투자 검토자) 70~80%, B타입(정보 수집자) 20~30%.

---

## 3. GSF Voice 5대 규칙 (KO 원고 기준)

### Rule 1 — 전문 용어: 첫 등장 시 괄호 해설 필수
```
❌ 공실률이 2%대라는 건
✅ 공실률(비어 있는 방의 비율)이 2%대라는 건
```

### Rule 2 — 한 문장 = 한 개념
콤마로 개념 3개 이상 연결 금지. 마침표로 끊고 다음 문장으로 이어간다.
```
❌ 수익률과 환율은 차가운 논리에 의해서만 움직이며, 감정이 개입할 여지가 없기 때문이다.
✅ 수익률은 냉정한 숫자로 움직입니다. 감정이 개입할 여지가 없습니다.
```

### Rule 3 — 1인칭 현장 경험: 포스트당 최소 1회
```
✅ "니혼바시에 직접 살면서 봤는데..."
✅ "이 물건 현장을 직접 갔을 때..."
✅ "한국과 일본 양쪽에서 투자를 실행해오면서..."
```

### Rule 4 — 독자적 의견: 포스트당 최소 1회
분석에서 결론으로 넘어갈 때 저자 입장을 명시한다.
```
✅ "시장은 이렇게 보지만, 제 판단은 다릅니다."
✅ "제 결론은 명확합니다."
```

### Rule 5 — 오프닝: 질문 또는 반전 훅
첫 문단에서 독자를 끌어당기는 문장으로 시작한다.
```
✅ "~라고 생각하시나요? 저는 반대로 봅니다."
✅ "보통 두 가지 반응이 나옵니다. (반응 A)라거나, (반응 B)라는 거죠."
```

## 4. 2-Pass 파이프라인

```
[KO 초안]
    ↓
[Pass 1] KO Voice Rewrite — 5대 규칙 적용
    ↓
[사용자 확인] — 재작성된 KO 검토 후 승인
    ↓
[Pass 2] EN/JA 번역 — voice-consistent 프롬프트
    ↓
[git push → Vercel 배포]
```

### Pass 1: KO Voice Rewrite 프롬프트 핵심 지침
- Rule 1~5 모두 적용
- 원문의 구조(H2/H3/표)는 유지
- 내용·사실 변경 금지 — 표현과 길이만 조정
- 출력: 완성된 KO 마크다운 (frontmatter 포함)

### Pass 2: EN 번역 프롬프트 핵심 지침
- 1인칭 `I`/`My` 필수 (기관체 `We` 금지)
- 뉴스레터 톤: 짧은 문장, 리드미컬한 리듬
- 전문 용어 첫 등장 시 em dash(`—`) 뒤에 한 줄 설명
- 예: `Cap Rate — the ratio of net operating income to property value`

### Pass 2: JA 번역 프롬프트 핵심 지침
- です/ます 경어체 필수 (だ/である 금지)
- 전문 용어: 일본어 번역 후 영어 원어 괄호 병기
- 예: `キャップレート(Cap Rate)`
- 신문 기사체 금지, 컨설턴트가 친절하게 설명하는 톤
